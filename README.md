# Cyclistic Bike-Share Case Study

## Project Overview
This project involves analyzing the usage patterns of Cyclistic bike-share data to identify key differences between casual riders and annual members. The insights derived from this analysis are intended to help the company design a new marketing strategy to convert casual riders into annual members.

## Tools Used
- **SQL:** PostgreSQL (for data cleaning and analysis)
- **Visualization:** Power BI

## Project Deliverables
- **[SQL Analysis Script](cyclistic_analysis.sql)**: Detailed code for data exploration and aggregation.
- **[Full Presentation](Cyclistic_Case_Study_Presentation.pdf)**: A comprehensive 8-slide summary of the business findings and recommendations.
- **[Dashboard Preview](cyclistic_dashboard_preview.png)**: A visual representation of the key metrics.

## Key Insights & Strategic Recommendations
- **Usage Patterns:** Annual members (3.7M rides) outperform casual riders (2.0M) in volume, while casual riders show longer trip durations (avg 20 mins vs 12 mins).
- **Weekly & Seasonal Trends:** Casual riders peak on weekends; usage for both groups surges in summer (June-August).
- **Strategic Recommendations:**
    - **Seasonal Marketing:** Targeted campaigns during peak summer months.
    - **Incentivized Conversion:** A "Weekend-to-Weekday" trial program to encourage daily commuting.
    - **Location-Based Outreach:** Deploying membership info at high-traffic leisure hubs.
    - **Value-Based Membership:** Highlighting long-term cost savings to frequent casual riders.
